ROUTEROS SENTINEL FLOW
# On-router scheduler polls Telegram
/system/scheduler → on-event
→ Build URL string (no /log, use :log)
→ :do { /tool/fetch url=$url } on-error { :log error $err }
→ Parse response JSON
→ Match command against allowlist
→ Execute via /system/script (NOT ssh-exec)
→ Send reply via :do { /tool/fetch url=$replyUrl }
 
# Critical conventions
✓ Spaces encoded as + in Telegram URLs
✓ on-event built as concatenated string variable
✓ All router access wrapped in :do/on-error
✗ Never use /log (use :log)
✗ Never use ssh-exec
✗ Never omit $ sigil on variables
